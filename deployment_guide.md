# LumBarong Deployment Guide (Hostinger)

This guide walks you through deploying LumBarong on **Hostinger** — the Laravel backend via **Hostinger Web Hosting (PHP)** and the Next.js frontend as a **static export** served from the same domain or a subdomain.

---

## Overview

| Component | Technology | Hosted On |
|-----------|-----------|-----------|
| Frontend  | Next.js (Static Export) | Hostinger (subdomain or subfolder) |
| Backend   | Laravel (PHP) | Hostinger (main domain or subdomain) |
| Database  | MySQL | Hostinger MySQL (built-in hPanel) |

---

## Step 1 — Set Up the Database on Hostinger

1. Log in to your **Hostinger hPanel**.
2. Go to **Hosting → Manage → Databases → MySQL Databases**.
3. Create a new database:
   - **Database name**: e.g., `u123456789_lumbarong`
   - **Username**: e.g., `u123456789_admin`
   - **Password**: Choose a strong password
4. Note down the following — you'll need them in the Laravel `.env`:
   - **DB_HOST**: `127.0.0.1`
   - **DB_PORT**: `3306`
   - **DB_DATABASE**: your full database name (e.g., `u123456789_lumbarong`)
   - **DB_USERNAME**: your full username (e.g., `u123456789_admin`)
   - **DB_PASSWORD**: your chosen password

> **Tip**: You can also import your existing `lumbarong_backup.sql` via **phpMyAdmin** (available in hPanel under Databases → phpMyAdmin).

## Step 2 — Deploy the Laravel Backend

Hostinger shared hosting supports PHP. You will upload Laravel's **public** folder contents into `public_html/` and everything else into a sibling `laravel/` folder.

### 2.1 Prepare Laravel Locally

In a terminal inside `backend_laravel/`:

```bash
composer install --optimize-autoloader --no-dev
php artisan key:generate
```

### 2.2 Configure `.env` for Production

Copy `.env.example` to `.env` and update the values:

```env
APP_NAME="LumBarong"
APP_ENV=production
APP_KEY=           # value generated by php artisan key:generate
APP_DEBUG=false
APP_URL=https://yourdomain.com

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=u123456789_lumbarong
DB_USERNAME=u123456789_admin
DB_PASSWORD=your_db_password

SESSION_DRIVER=database
CACHE_STORE=database
QUEUE_CONNECTION=database

FILESYSTEM_DISK=public

# CORS — set to your frontend URL
FRONTEND_URL=https://yourdomain.com

# Optional — for password reset emails via Hostinger SMTP
MAIL_MAILER=smtp
MAIL_HOST=smtp.hostinger.com
MAIL_PORT=587
MAIL_USERNAME=your@email.com
MAIL_PASSWORD=your_email_password
MAIL_FROM_ADDRESS=your@email.com
MAIL_FROM_NAME="LumBarong"
```

### 2.3 Upload Files via File Manager or FTP

Use this folder layout on Hostinger:

```
/home/u123456789/domains/yourdomain.com/
├── public_html/               ← Contents of backend_laravel/public/ go HERE
│   ├── index.php
│   ├── .htaccess
│   └── ...
└── laravel/                   ← All other Laravel files go HERE
    ├── app/
    ├── bootstrap/
    ├── config/
    ├── database/
    ├── routes/
    ├── storage/
    ├── vendor/
    ├── .env
    └── ...
```

**Steps:**
1. In hPanel, open **File Manager** (or connect via FTP using FileZilla).
2. Upload everything **except** the `public/` folder into a new folder named `laravel/` (next to `public_html/`).
3. Upload the **contents** of `backend_laravel/public/` into `public_html/`.
4. Edit `public_html/index.php` — update the two paths so Laravel can find its files:

```php
// Change these two lines in public_html/index.php:
require __DIR__.'/../laravel/vendor/autoload.php';
$app = require_once __DIR__.'/../laravel/bootstrap/app.php';
```

### 2.4 Fix Storage Permissions

1. In File Manager, navigate to `laravel/storage/` and `laravel/bootstrap/cache/`.
2. Right-click each folder → **Change Permissions** → set to **755**.
3. If your plan includes **SSH access** (hPanel → Advanced → SSH Access), run:
   ```bash
   cd /home/u123456789/domains/yourdomain.com/laravel
   php artisan storage:link
   ```
   Otherwise, manually create a symbolic link so that `public_html/storage` points to `laravel/storage/app/public`.

### 2.5 Run Migrations

**Via SSH terminal (recommended):**
```bash
cd /home/u123456789/domains/yourdomain.com/laravel
php artisan migrate --force
php artisan db:seed --force   # only if you have seeders
```

**Without SSH:** Import `lumbarong_backup.sql` directly via **phpMyAdmin** in hPanel.

### 2.6 Verify `.htaccess` in `public_html/`

Ensure the default Laravel `.htaccess` exists in `public_html/`:

```apache
<IfModule mod_rewrite.c>
    <IfModule mod_negotiation.c>
        Options -MultiViews -Indexes
    </IfModule>

    RewriteEngine On

    # Force HTTPS
    RewriteCond %{HTTPS} off
    RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]

    # Handle Authorization Header
    RewriteCond %{HTTP:Authorization} .
    RewriteRule .* - [E=HTTP_AUTHORIZATION:%{HTTP:Authorization}]

    # Redirect Trailing Slashes If Not A Folder
    RewriteCond %{REQUEST_FILENAME} !-d
    RewriteCond %{REQUEST_URI} (.+)/$
    RewriteRule ^ %1 [L,R=301]

    # Send Requests To Front Controller
    RewriteCond %{REQUEST_FILENAME} !-d
    RewriteCond %{REQUEST_FILENAME} !-f
    RewriteRule ^ index.php [L]
</IfModule>
```

## Step 3 — Deploy the Next.js Frontend (Static Export)

The Next.js project is already configured for static export (`output: 'export'`). The built `out/` folder is plain HTML/CSS/JS that Hostinger can serve directly.

### 3.1 Build the Frontend Locally

In a terminal inside `frontend/`:

1. Create a `.env.local` file with your production backend URL:
   ```env
   NEXT_PUBLIC_API_URL=https://yourdomain.com/api
   ```
2. Build:
   ```bash
   npm install
   npm run build
   ```
   This generates the `frontend/out/` folder.

### 3.2 Upload the Static Files

**Option A — Subdomain (Recommended): `app.yourdomain.com`**
1. In hPanel → **Domains → Subdomains**, create `app.yourdomain.com`.
2. Set its document root to a new folder, e.g., `public_html/app/`.
3. Upload all contents of `frontend/out/` into that folder.

**Option B — Subfolder: `yourdomain.com/app`**
1. In File Manager, create a folder `public_html/app/`.
2. Upload all contents of `frontend/out/` into `public_html/app/`.

### 3.3 Handle Client-Side Routing

Next.js static exports use client-side routing — direct URL access (e.g., refreshing the page) will 404 without this fix. Add an `.htaccess` file inside the frontend folder:

```apache
<IfModule mod_rewrite.c>
    RewriteEngine On
    RewriteBase /

    # Don't rewrite real files or directories
    RewriteCond %{REQUEST_FILENAME} -f [OR]
    RewriteCond %{REQUEST_FILENAME} -d
    RewriteRule ^ - [L]

    # Rewrite everything else to index.html
    RewriteRule ^ index.html [L]
</IfModule>
```

## Step 4 — Configure CORS on the Backend

The Laravel backend must allow requests from your frontend domain. Open `backend_laravel/config/cors.php` and update `allowed_origins`:

```php
'allowed_origins' => ['https://app.yourdomain.com'],
```

This should already be driven by `FRONTEND_URL` in `.env` — confirm that `config/cors.php` references it.

---

## Step 5 — Enable HTTPS (Free SSL)

1. In hPanel, go to **SSL** under your domain.
2. Enable the **Free SSL (Let's Encrypt)** certificate for your main domain.
3. Do the same for `app.yourdomain.com` (if using a subdomain for the frontend).
4. The Force HTTPS rule is already included in the `.htaccess` above (Step 2.6).

## Step 6 — Final Deployment Checklist

- [ ] MySQL database created in hPanel with correct credentials
- [ ] `backend_laravel/` files uploaded: non-public files in `laravel/`, public files in `public_html/`
- [ ] `public_html/index.php` paths updated to point to `laravel/` folder
- [ ] Storage and cache permissions set to `755`
- [ ] `storage:link` created (via SSH or manual symlink)
- [ ] Migrations run (via SSH `php artisan migrate --force` or phpMyAdmin import)
- [ ] Frontend built locally with correct `NEXT_PUBLIC_API_URL`
- [ ] `frontend/out/` contents uploaded to subdomain or subfolder
- [ ] `.htaccess` added inside the frontend folder for client-side routing
- [ ] CORS `allowed_origins` updated in Laravel with the frontend URL
- [ ] Free SSL enabled for both backend domain and frontend subdomain

## Troubleshooting

| Problem | Solution |
|---------|----------|
| `500 Internal Server Error` | Check `laravel/storage/logs/laravel.log`. Temporarily set `APP_DEBUG=true` in `.env`. |
| Blank page on frontend | Check browser console for JS errors. Verify `NEXT_PUBLIC_API_URL` is correct and the backend is reachable. |
| API calls failing (CORS error) | Verify `FRONTEND_URL` in `.env` and `config/cors.php` `allowed_origins`. |
| Images not loading | Ensure `storage:link` is set up and `FILESYSTEM_DISK=public` is in `.env`. |
| 404 on page refresh (frontend) | Make sure the frontend `.htaccess` is uploaded alongside `index.html`. |
| Database connection refused | Double-check DB credentials in `.env`. Use `127.0.0.1`, not `localhost`. |
| `php artisan` not available | Use Hostinger SSH Access (hPanel → Advanced → SSH Access) or upgrade your plan. |
